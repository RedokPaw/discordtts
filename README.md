### Usage (only docker compose deployment)
pull to your server
```bash
git pull && mv env.example .env
```
And add your discord token bot in .env, then:
```bash
docker compose up -d && docker compose logs -f -t
```

Из минусов: бот не закрепляется за каналом, из вашего канала его может "украсть" обычным джоином любой другой пользователь сервера. Пока что не хватает кастомизации (увеличить количество паралелльных процессов ffmpeg например). С первых версий намного улучшена работа в условиях работы во множестве серверов, устранены утечки памяти и сообщения теперь обрабатывается мультипоточно. Код все еще пишется, поэтому многая логика "жесткая", потом добавлю интерфейсов. 

Если вдруг это читает кто-то кроме меня (или вспомнить самому мне же), сейчас стоит довольно маленький лимит памяти в Dockerfile. Если планируется больеш чем 2 сервера, нужно увеличить лимит Xmx в Dockerfile.
